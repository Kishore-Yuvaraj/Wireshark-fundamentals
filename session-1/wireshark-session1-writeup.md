# Wireshark Session 1: Live Traffic Capture & Analysis

📅 Date: April 20, 2026  
🎯 Objective: Capture and analyze real DNS, HTTP, and TCP traffic using Wireshark  
🛠️ Tools: Wireshark 4.x (Windows)

---

## 🚀 Overview

In this session, I captured and analyzed live network traffic from my machine to understand how core protocols (DNS, HTTP, TCP) behave in real-world conditions.

- Total packets captured: ~6,688  
- Traffic generated by visiting: Google, GitHub, YouTube  
- Focus: DNS resolution, HTTP communication, TCP handshake  

---

## 🌐 DNS Analysis

**Filter Used:** `dns`  
**Packets Observed:** 4 (2 queries + 2 responses)

### Key Findings:
- DNS queries were sent from my machine to the resolver
- Response returned IP addresses:
  - 8.8.8.8
  - 8.8.4.4
- TTL observed: ~8 minutes

### 🔐 SOC Insight:
- TTL values help detect suspicious domains (very low TTL = possible DNS tunneling)
- Multiple IPs can indicate load balancing or fast-flux malware
- Monitoring DNS queries helps detect malicious domain communication

---

## 🌍 HTTP Analysis

**Filter Used:** `http`  
**Packets Observed:** 6

### Flow Observed:
1. Client request: `GET /online`
2. Server response: `301 Moved Permanently`
3. Client follows redirect: `GET /online/`
4. Server response: `200 OK`
5. Browser automatically requests `/favicon.ico`

### 🔐 SOC Insight:
- Redirect chains can hide malicious destinations
- HTTP traffic is unencrypted and can expose sensitive data
- Favicon requests are normal browser behavior and should not be misinterpreted

---

## 🔗 TCP 3-Way Handshake

**Packets Identified:** 1449, 1450, 1451

### Handshake Process:
- SYN → Client initiates connection
- SYN-ACK → Server acknowledges request
- ACK → Connection established

### 🔐 SOC Insight:
- Incomplete handshake may indicate SYN flood attack
- SYN + RST patterns can indicate port scanning
- Monitoring TCP flags helps identify abnormal network behavior
